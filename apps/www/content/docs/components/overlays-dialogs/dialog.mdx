## Basic Usage
```tsx
import {
  Dialog,
  DialogClose,
  DialogDescription,
  DialogFooter,
  DialogHeader,
  DialogContent,
  DialogTitle,
  DialogTrigger,
} from "@/components/ui/dialog";

export function DialogDemo() {
  return (
    <Dialog>
      <DialogTrigger>Show Alert Dialog</DialogTrigger>
      <DialogContent showCloseButton>
        <DialogHeader>
          <DialogTitle>Are you absolutely sure?</DialogTitle>
          <DialogDescription>
            This action cannot be undone. This will permanently delete your
            account and remove your data from our servers.
          </DialogDescription>
        </DialogHeader>
        <DialogFooter>
          <DialogClose>Cancel</DialogClose>
          <DialogClose>Continue</DialogClose>
        </DialogFooter>
      </DialogContent>
    </Dialog>
  );
}
```

## Variants

The `DialogContent` layout determines the visual style and entry animation:

- `responsive` (default): Bottom sheet on mobile, centered modal on desktop.
- `center`: Centered modal with fade-up animation.
- `top`: Top-aligned modal with fade-down animation.
- `scrollable`: Optimized for containers with internal scrolling.
- `stacked`: Shifts and scales for clean nested dialog flows.
- `element-outside`: For UIs with elements positioned outside the main container.

## Cookbook

### Scrolling

#### Internal Scrolling
Set `DialogContent` to `layout="scrollable"`. Use `ScrollArea` or `overflow-y-auto` for the content. Elements outside the scrollable area remain fixed.

<ComponentPreview name="dialog-inside-scroll" />

#### Elements Outside
Use `layout="element-outside"` for designs where controls should live outside the popup.

<ComponentPreview name="dialog-elements-outside" />

#### Viewport Scrolling
For very long content, use `DialogViewport` as the scrolling container. This allows the entire dialog to scroll with the page.

<ComponentPreview name="dialog-outside-scroll" />

### Close Confirmation
Use `layout="stacked"` for both `DialogContent` and [`AlertDialogContent`](/docs/components/overlays-dialogs/alert-dialog#composite-component) to handle nested confirmation flows gracefully.

<ComponentPreview name="dialog-close-confirmation" />

### Controlled State
Manage the `open` state manually for external triggers or async closing logic.

```tsx title="Controlled dialog"
const [open, setOpen] = React.useState(false);

<Dialog open={open} onOpenChange={setOpen}>
  <DialogTrigger>Open</DialogTrigger>
  <DialogContent>
    <form onSubmit={async () => {             
      await submitData();             
      setOpen(false);   
    }}>
      ...
    </form>
  </DialogContent>
</Dialog>
```

#### From Dropdown Menu
Use the controlled state when opening a dialog from a [`DropdownMenu`](/docs/components/overlays-dialogs/dropdown-menu) to avoid focus and nesting issues.

<ComponentPreview name="dialog-from-menu" />

### Detached Triggers & Payloads
Use `createDialogHandle` to link a `DialogTrigger` to a `Dialog` when they cannot be nested within the same root.

```tsx title="Detached trigger"
const demoDialog = createDialogHandle();

<DialogTrigger handle={demoDialog}>Open</DialogTrigger>
<Dialog handle={demoDialog}>...</Dialog>
```

A single dialog can handle multiple triggers and render content dynamically based on a payload.

## API reference

### Primitives

| Component | Description |
| :--- | :--- |
| [`Dialog`](https://base-ui.com/react/components/dialog#root) | Groups all parts of the dialog. Doesnâ€™t render its own HTML element. |
| [`DialogTrigger`](https://base-ui.com/react/components/dialog#trigger) | A button that opens the dialog. |
| [`DialogPortal`](https://base-ui.com/react/components/dialog#portal) | A portal element that moves the popup to a different part of the DOM. Renders a `<div>` element.|
| [`DialogBackdrop`](https://base-ui.com/react/components/dialog#backdrop) | An overlay displayed beneath the popup. Renders a `<div>` element. |
| [`DialogViewport`](https://base-ui.com/react/components/dialog#viewport) | A positioning container for the dialog popup that can be made scrollable. Renders a `<div>` element. |
| [`DialogPopup`](https://base-ui.com/react/components/dialog#popup) | A container for the dialog contents. Must be used inside `DialogPortal`. Renders a `<div>` element. |
| `DialogHeader` | Styled container for convenience purpose. Renders a `<div>` element. |
| [`DialogTitle`](https://base-ui.com/react/components/dialog#title) | A heading that labels the dialog. Renders an `<h2>` element. |
| [`DialogDescription`](https://base-ui.com/react/components/dialog#description) | A paragraph with additional information about the dialog. Renders a `<p>` element. |
| `DialogFooter` | Styled container for convenience purpose. Renders a `<div>` element. |
| [`DialogClose`](https://base-ui.com/react/components/dialog#close) | A button that closes the dialog. Renders a `<button>` element.|

### Composite Components

| Component | Description |
| :--- | :--- |
| `DialogContent` | **Standard usage**. Groups Portal, Backdrop, Viewport, and Popup. |
| `DialogElementOutsideContent` | Extension of `DialogContent` for UIs with elements outside the popup. |

### DialogContent Layout Variants

| Variant | Description |
| :--- | :--- |
| `responsive` | Mobile bottom sheet; Desktop centered modal. |
| `center` | Centered modal with fade-up transition. |
| `top` | Top-aligned modal with fade-down transition. |
| `scrollable` | Fixed header/footer layout for internal scrolling. |
| `stacked` | Scaling and offset support for nested dialogs. |
| `element-outside` | Supports interactive elements outside the popup box. |
