## Cookbook

### Rendering as another tag

Base UI uses the `render` prop to change the underlying DOM element while preserving accessibility and behavior. This replaces the `asChild` pattern found in Radix UI.

```tsx /render/ /nativeButton/
<Button render={<div />} nativeButton={false}>
  Button that can contain complex children
</Button>
```

Here's an example rendering a link that looks like a button.

```tsx /render/ /nativeButton/
import Link from "next/link"
import { Button } from "@/components/ui/button"

export function ButtonLink() {
  return (
    <Button render={<Link href="/login" />} nativeButton={false}>
      Login
    </Button>
  )
}
```

<Callout variant="warning" title="Warning">
Be sure to set `nativeButton={false}` to ensure proper accessibility semantics if you are rendering a non-button element.
</Callout>
### Loading State

The `isLoading` prop automatically disables the button and prevents interaction.

<Callout variant="info" title="Accessibility Note">
**Unlike a standard `disabled` attribute, `isLoading` preserves focus on the button (via `focusableWhenDisabled`). This prevents focus from being lost to the `body` during async form submissions, maintaining a smooth keyboard navigation flow.**
</Callout>

### Cursor

Tailwind v4 switched from `cursor: pointer` to `cursor: default` for the button component.

If you want to keep the `cursor: pointer` behavior, add the following code to your CSS file:

```css
@layer base {
  button:not(:disabled),
  [role="button"]:not(:disabled) {
    cursor: pointer;
  }
}
```

## API Reference

| Prop | Type | Description |
| :--- | :--- | :--- |
| `variant` | `default` \| `destructive` \| `outline` \| `secondary` \| `ghost` \| `link` \| `glow` \| `unstyled` | The visual style of the button. |
| `size` | `default` \| `sm` \| `lg` \| `icon` \| `icon-sm` \| `icon-lg` \| `icon-xs` | The size of the button. `icon-*` variants are square. |
| `isLoading` | `boolean` | `false` | Sets the button to a visual loading state, disables interaction, but maintains keyboard focusability. |
| [`render`](https://base-ui.com/react/components/button#Button-render) | `ReactElement` | Replaces the underlying HTML element. |
|  [`nativeButton`](https://base-ui.com/react/components/button#Button-nativeButton) | `boolean` | When `false`, the button will be rendered as another tag. |
|  [`focusableWhenDisabled`](https://base-ui.com/react/components/button#Button-focusableWhenDisabled) | `boolean` | Whether the button should be focusable when disabled. |
